## Design Standards

I had the following conditions when deciding to build my blogging site:

1. Utilise some sort of Infrastucture as Code (IaC).
    - As I have experience in [CDK](https://aws.amazon.com/cdk/) I've been really interested in expanding my knowledge in IaC. So I'd like to create something that's maintainable but also educational.
2. My blog entries will be written in markdown.
    - I like writing in markdown, it's simple and usable across different systems.

Ideally the flow would be as follows:

- Blog commits go to GitHub repository (Markdown + Hugo config)

- GitHub Actions triggers Hugo build → generates static site

- Deploys to S3 bucket (static website hosting enabled)

- CloudFront uses S3 website endpoint as origin which allows me to serve my blog globally via CDN.

- Users access blog via domain that I already own.

> Note: I could use CodePipeline and CodeBuild that will trigger on Github commits and then build my Hugo site using CodeBuild. But for now, I'll stick to GitHub actions and look into CodePipeline + CodeBuild later - maybe I'll create a new blog entry for this...

## Using Hugo

Reading the documentation for Hugo, it looks like I'll need to download this tool. I'm not a big fan of downloading stuff onto my devices (unless absolutely necessary) so I figured maybe use a virtual machine to deploy. Then again, it seemed quite overkill to do it this way so I've gone ahead and decided to containerize it using Docker.
